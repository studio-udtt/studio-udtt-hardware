# studio-udtt-hardware
'공간의 숨결 시각화' 프로젝트의 데이터를 수집하고 처리하는 IoT 하드웨어 시스템입니다.

---

## 하드웨어 구성 및 설계 
ESP32-S3 기반의 하드웨어로 총 4개의 센서를 활용하여 데이터를 수집합니다.

| 센서 유형 | 모델 | 상세 역할 |
| :--- | :--- | :--- |
| ToF 센서 | VL53L0X | 유동 인구 계측 및 45도 각도 설치를 통한 입/퇴장 판단 |
| mmWave 센서 | S3KM1110 | 재실 유무 및 정적/동적 활동 상태 판별  |
| 음향 센서 | MAX9814 | 소음 수준 측정을 통한 청각적 역동성 분석  |
| 조도 센서 | BH1750 | 공간의 시각적 환경 상태 파악을 통한 상태 파악 |

## 설계 파이프라인
모든 센서 데이터는 데이터 수집부터 시각화까지 4단계의 설계 파이프라인을 구축한다.
  1. **[데이터 수집]** 센서 기반 IoT 하드웨어를 통해 비가시적 환경 데이터와 유동 인구 데이터 실시간 포착
  2. **[정규화]** 서로 다른 형태의 변수를 분석 가능한 형태로 정제 및 정규화
  3. **[분류]** 정제된 데이터를 알고리즘에 적용하여 공간 활성화 정도 분류
  4. **[시각화]** 분류 및 가공된 데이터를 참여자에게 실시간으로 전달

## 알고리즘 및 데이터 처리
* **ToF 레이저 거리 센서 [VL53L0X]**
  * 노이즈 필터링: 8190 등으로 포화되는 데이터 대신 유효 반경(2000mm) 내 데이터만 추출
  * 값 치환: 비정상 데이터 유입 시 학습된 기준 거리(1500mm)로 강제 치환하여 오차 차단 
* **mmWave 센서 [S3KM1110]**
  * 실시간 데이터 버터링: 메인 루프를 통해 수신 버퍼를 실시간 파싱하여 CPU 부하 최소화 및 데이터 유실 방지
  * 유효 움직임 판단: 직전 측정값과 현재값의 절대 변위량이 0.1m 이상일 때만 '유효 움직임'으로 간주
  * 안정성 유지: 정지 상태로 전환 시 2초의 유예 시간을 제공하여, 시스템의 활성화/비활성화 반복 현상 방
* **음향 센서 [MAX9814]**
  * 고속 샘플링: 0.05초의 짧은 주기로 신호를 집중 수집하여 데이터 왜곡 최소화
  * Peak-to-Peak 판정: 최댓값과 최솟값의 차이를 계산하여 소리의 세기를 도출하며, 1000 이상의 값은 노이즈로 판별하여 무시
  * 공간 상태 분류: 수집된 값에 따라 4단계(정적 <100, 조용함 <150, 적당함 ≤400, 활발함 <1000)로 청각적 활성도 분류
* **조도 센서 [BH1750]**
  * 환경 상태 판별: 80lx를 임계치로 설정하여 실내 조명 가동 여부 및 공간의 운영 상태를 수치화
  * 공간 상태 분류: 80lx 이상은 '밝음: 주간/활성화 상태', 80lx 미만은 '어두움: 야간/소등 상태'로 판정하여 환경적 활성도 분

## 공간의 숨결 지수 정규화 로직
서로 다른 단위를 가진 센서 데이터를 0~100점 범위로 정규화한 뒤, 아래와 같이 가중치를 결합하여 최대 100점인 공간의 숨결 지수를 정의합니다.
| 항목 | 가중치 | 근거 |
| :--- | :--- | :--- |
| 인원 점수 | 40% | 공간 활성도를 결정하는 가장 직관적인 지표 |
| 소음 점수 | 40% | 공간 내부 활동의 청각적 역동성 |
| 조도 점수 | 20% | 공간의 환경적 활성도 반영   |
| 움직임 가산점 | +10점 | 움직임 포착 시 활력 점수 부여 |
