# studio-udtt-hardware
'공간의 숨결 시각화' 프로젝트의 데이터를 수집하고 처리하는 IoT 하드웨어 시스템입니다.

---

## 하드웨어 구성 및 설계 
ESP32-S3 기반의 하드웨어로 총 4개의 센서를 활용하여 데이터를 수집합니다.

| 센서 유형 | 모델 | 상세 역할 |
| :--- | :--- | :--- |
| ToF 센서 | VL53L0X | 유동 인구 계측 및 45도 각도 설치를 통한 입/퇴장 판단 |
| mmWave 센서 | S3KM1110 | 재실 유무 및 정적/동적 활동 상태 판별  |
| 음향 센서 | MAX9814 | 소음 수준 측정을 통한 청각적 역동성 분석  |
| 조도 센서 | BH1750 | 공간의 시각적 환경 상태 파악을 통한 상태 파악 |

## 설계 파이프라인
모든 센서 데이터는 데이터 수집부터 시각화까지 4단계의 설계 파이프라인을 구축합니다.
  1. **[데이터 수집]** 센서별 환경/인원 데이터 실시간 포착
  2. **[정규화]** 서로 다른 데이터 변수를 분석 가능한 형태로 정제 및 정규화
  3. **[분류]** 알고리즘 적용을 통한 공간 활성화 정도 분류
  4. **[시각화]** 참여자에게 실시간 상태 정보 전달

## 알고리즘 및 데이터 처리
* **ToF 레이저 거리 센서 [VL53L0X]**
  * 노이즈 필터링: 8190 등 포화 데이터 배제, 유효 반경(2000mm) 내 데이터만 추출
  * 값 치환: 비정상 데이터 유입 시 기준 거리(1500mm)로 강제 치환하여 오차 방지 
* **mmWave 센서 [S3KM1110]**
  * 데이터 버터링: 메인 루프에서 수신 버퍼 실시간 파싱(CPU 부하 최소화)
  * 유효 움직임 판단: 직전 값 대비 절대 변위량이 0.1m 이상일 때만 '유효 움직임'으로 간주
  * 안정성 유지: 정지 전환 시 2초 유예 시간 제공(시스템 반복 활성화 방지)
* **음향 센서 [MAX9814]**
  * 고속 샘플링: 0.05초 주기의 신호 수집으로 데이터 왜곡 최소화
  * Peak-to-Peak 판정: 최댓값-최솟값의 차이 산출 (1000 이상의 값은 노이즈로 무시)
  * 공간 상태 분류: 청각적 역독성 4단계 분류
    * 정적 <100 / 조용함 <150 / 적당함 ≤400 / 활발함 <1000
* **조도 센서 [BH1750]**
  * 환경 상태 판별: 80lx를 임계치로 실내 조명 및 운영 상태 수치화
  * 공간 상태 분류:
    * 밝음(≥80lx): 주간 및 활성화 상태
    * 어두움(<80lx): 야간 및 소등 상태


## 공간의 숨결 지수 정규화 로직
수집된 센서 데이터를 아래 가중치에 따라 0~100점 범위로 정규화합니다.
| 항목 | 가중치 | 근거 |
| :--- | :--- | :--- |
| 인원 점수 | 40% | 공간 활성도를 결정하는 가장 직관적인 지표 |
| 소음 점수 | 40% | 공간 내부 활동의 청각적 역동성 |
| 조도 점수 | 20% | 공간의 환경적 활성도 반영   |
| 움직임 가산점 | +10점 | 움직임 포착 시 활력 점수 부여 |

$$숨결 = (인원 \times 0.4) + (소음 \times 0.4) + (조도 \times 0.2) + 움직임 (10)$$
